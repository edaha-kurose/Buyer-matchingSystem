## 0. 背景と問題意識

（以下、ユーザー指定テキストを原文ママで保持）

勝者やバイヤー、もしくは通常の企業で提案を受ける側が、たくさんの商品やサービスを自社の買う側担当者がプレゼンを聞いて内容判断し、社内で採用可否を議論している。

しかし商社のように大量の商品・サービスを募集する立場では、サプライヤーからのプレゼンを大量に聞かなければならず、1件30分〜1時間×数十件という**深刻な時間拘束問題**が発生している。

この問題を解決するために、
- バイヤー側はAIを通じてプレゼンを受付
- バイヤーが事前に「好み・条件・興味」を設定
- AIが一次スクリーニングを行い、条件に合う提案のみを人が確認

という仕組みをAIで構築する。

## 1.5. 資料ベースプレゼン機能（非同期Q&Aモード）

### 概要

動画やリアルタイムプレゼンに加え、**資料提出のみ**でプレゼンを完結できるモードを提供する。
サプライヤー側の負担を軽減しつつ、AIによる自動質問→回答の往復で必要情報を収集する。

### コンセプト

> **「資料を出すだけでプレゼン完了」- サプライヤーファーストの非同期プレゼン**

### 資料ベースプレゼンのメリット

| ステークホルダー | メリット |
|-----------------|---------|
| サプライヤー | 時間拘束なし、好きなタイミングで回答可能、準備コスト削減 |
| バイヤー | 大量の提案を効率的に処理、均一な基準で評価可能 |
| システム | リアルタイム処理不要、技術的難易度低、コスト削減 |

```python
# 資料解析パイプライン（疑似コード）

class DocumentAnalysisPipeline:
    """資料ベースプレゼンの解析パイプライン"""
    
    async def process_submission(self, proposal_id: str) -> AnalysisResult:
        # 1. 資料のテキスト抽出
        documents = await self.document_repo.get_by_proposal(proposal_id)
        extracted_texts = []
        
        for doc in documents:
            if doc.type == "pdf":
                text = await self.pdf_extractor.extract(doc.path)
            elif doc.type == "pptx":
                text = await self.pptx_extractor.extract(doc.path)
            elif doc.type == "image":
                text = await self.ocr_service.extract(doc.path)
            extracted_texts.append({
                "filename": doc.filename,
                "content": text
            })
        
        # 2. 情報抽出（LLM）
        extracted_info = await self.extraction_agent.extract(
            documents=extracted_texts,
            schema=self.info_schema
        )
        
        # 3. バイヤー条件との照合
        buyer_config = await self.config_repo.get_active_config(proposal.buyer_id)
        
        # 4. Gap分析 & 質問生成
        gap_analysis = await self.gap_agent.analyze(
            extracted_info=extracted_info,
            buyer_requirements=buyer_config.requirements
        )
        
        questions = await self.question_agent.generate(
            gaps=gap_analysis.gaps,
            priority=gap_analysis.priority,
            max_questions=5  # 初回は最大5問
        )
        
        return AnalysisResult(
            proposal_id=proposal_id,
            extracted_info=extracted_info,
            gap_analysis=gap_analysis,
            questions=questions
        )
```

### 回答評価ロジック

```python
# 回答評価ロジック（疑似コード）

class AnswerEvaluator:
    """サプライヤー回答の評価"""
    
    async def evaluate_round(
        self, 
        questions: List[Question], 
        answers: List[Answer]
    ) -> RoundEvaluation:
        
        evaluations = []
        
        for q, a in zip(questions, answers):
            # 回答の質を評価
            eval_result = await self.llm.evaluate(
                prompt=f"""
                以下の質問と回答を評価してください。
                
                質問: {q.text}
                質問の意図: {q.intent}
                期待する情報: {q.expected_info}
                
                回答: {a.text}
                
                評価基準:
                1. 質問に対して直接回答しているか
                2. 具体的な数値・事実が含まれているか
                3. 追加の確認が必要か
                
                JSON形式で回答:
                {{
                    "answered": true/false,
                    "completeness": 0-100,
                    "extracted_facts": [...],
                    "needs_followup": true/false,
                    "followup_reason": "..."
                }}
                """
            )
            evaluations.append(eval_result)
        
        # 全体の充足度を計算
        total_completeness = sum(e.completeness for e in evaluations) / len(evaluations)
        
        # 追加ラウンドが必要か判定
        needs_another_round = (
            total_completeness < 80 and  # 充足度80%未満
            any(e.needs_followup for e in evaluations)  # フォローアップ必要な質問あり
        )
        
        return RoundEvaluation(
            evaluations=evaluations,
            total_completeness=total_completeness,
            needs_another_round=needs_another_round
        )
```

## 8.5. フィルタリングロジック詳細設計

### フィルタリング実装（疑似コード）

```python
class ProposalFilter:
    """提案フィルタリングエンジン"""
    
    def __init__(self, buyer_config: BuyerConfig):
        self.config = buyer_config
        self.mandatory_checker = MandatoryChecker(buyer_config.mandatory_conditions)
        self.scorer = CategoryScorer(buyer_config.weights)
    
    async def filter_proposal(self, proposal: Proposal) -> FilterResult:
        """単一提案のフィルタリング"""
        
        # Stage 1: 必須条件チェック
        mandatory_result = self.mandatory_checker.check(proposal.extracted_info)
        
        if not mandatory_result.passed:
            return FilterResult(
                proposal_id=proposal.id,
                classification="rejected",
                reason=f"必須条件不適合: {mandatory_result.failed_conditions}",
                score=0,
                requires_human_review=False
            )
        
        # Stage 2: スコアリング
        scores = self.scorer.calculate(proposal.extracted_info, proposal.qa_history)
        total_score = sum(
            scores[cat] * self.config.weights[cat] 
            for cat in scores
        )
        
        # Stage 3: 分類
        classification = self._classify(total_score, proposal)
        
        return FilterResult(
            proposal_id=proposal.id,
            classification=classification,
            score=total_score,
            category_scores=scores,
            requires_human_review=(classification in ["candidate", "consider"]),
            ai_summary=await self._generate_summary(proposal, scores)
        )
    
    def _classify(self, score: float, proposal: Proposal) -> str:
        """スコアに基づく分類"""
        if score >= 80:
            return "candidate"  # 採用候補
        elif score >= 60:
            return "consider"   # 検討
        elif score >= 40:
            return "hold"       # 保留
        else:
            return "rejected"   # 不採用
    
    async def _generate_summary(self, proposal: Proposal, scores: dict) -> str:
        """AI要約生成"""
        return await self.llm.generate(
            prompt=f"""
            以下の提案について、バイヤー向けの簡潔な要約を生成してください。
            
            提案内容: {proposal.extracted_info}
            Q&A履歴: {proposal.qa_history}
            スコア: {scores}
            
            要約に含めるべき内容:
            1. 提案の概要（2-3文）
            2. 主な強み
            3. 懸念点・確認事項
            4. 推奨アクション
            """
        )
```
